Getting started

Using the main features of Calimero Network involves several components and processes tailored to developers looking to build decentralized, privacy-focused applications. Here’s an overview of how to engage with its core functionalities, along with examples of common use cases:

1. Setting Up Client Nodes​

  • Purpose: Client nodes are the entry points for applications to connect to the Calimero Network, facilitating the execution of WASM applications in a peer-to-peer (P2P) environment.
  • Common Use Case: A developer creating a decentralized messaging app would set up a client node to handle encrypted messages between users.
  • How to Use: 
    # Example command to initialize a client node
    calimero_node init --name myNode --network calimeroNetwork

2. Developing with Protocol SDK (Rust Library)​

  • Purpose: The Protocol SDK assists in defining the rules for application-specific networks, automating code generation to simplify the development process.
  • Common Use Case: Defining the communication protocol for a decentralized finance (DeFi) application, including smart contract interactions and P2P transactions.
  • How to Use: 
    // Example Rust code snippet to define a protocol
    use calimero_sdk::ProtocolBuilder;

    let my_protocol = ProtocolBuilder::new()
        .name("DeFiProtocol")
        .version("1.0")
        .build();

3. Utilizing Client SDKs​

  • Purpose: Client SDKs provide interfaces for interacting with client nodes, managing session keys, and handling authentication and encryption.
  • Common Use Case: A mobile app developer integrating Calimero’s encryption for secure user communications within the app.
  • How to Use: 
    // Example JavaScript code using Client SDK
    import { CalimeroClient } from 'calimero-client-sdk';

    const client = new CalimeroClient({ nodeUrl: 'https://node.calimero.network' });
    client.authenticateUser('userToken').then(() => {
      console.log('User authenticated successfully');
    });

4. Interacting with the Specialized Compute Market​

  • Purpose: The specialized compute market allows nodes to offer or request specific computational resources or services, such as data replication or executing complex algorithms.
  • Common Use Case: An application requiring heavy computational tasks like rendering 3D graphics in a decentralized game can request resources from the specialized compute market.
  • How to Use: 
    # Example command to request specialized computation
    calimero_compute request --type rendering --app my3DGameApp

5. End-to-End Encryption with Double Ratchet Algorithm​

  • Purpose: Ensuring secure, private communication between peers in the network.
  • Common Use Case: A chat application using the Double Ratchet Algorithm to encrypt messages, ensuring that each message has a unique encryption key.
  • How to Use:
    • Typically, the use of the Double Ratchet Algorithm is abstracted within the Client SDKs, requiring minimal direct interaction from the developer.

Examples of Commands and Options​

While specific commands and options would depend on the implementation of the Calimero Network tools and SDKs, here are some hypothetical examples:

  • Initializing a project: calimero init myProject
  • Deploying an application to the network: calimero deploy --app myDapp
  • Managing keys and authentication: calimero keys generate --app myDapp

By integrating these components, developers can build decentralized applications that prioritize user privacy and data ownership, from secure messaging platforms to privacy-preserving financial applications. The Calimero Network offers a comprehensive suite of tools and services to facilitate this, all while maintaining an open-source ethos and fostering a community of privacy-centric developers and users.
